# Архитектура проекта — Калькулятор РЖД

Документ описывает архитектуру системы "Калькулятор норм расхода электроэнергии РЖД" с использованием C4-модели.

---

## C1: Context (Контекст системы)

### Границы системы

```
┌─────────────────────────────────────────────────────────────┐
│                    Калькулятор РЖД                          │
│  (Веб-приложение для расчета энергопотребления поездов)     │
└─────────────────────────────────────────────────────────────┘
```

### Пользователи (Actors)

| Пользователь | Описание | Взаимодействие |
|--------------|----------|----------------|
| **Оператор** | Сотрудник РЖД, выполняющий расчеты | Выбирает маршрут, вводит параметры поезда, получает результат расчета |
| **Администратор** | Добавляет новые маршруты | Создает .md файлы с данными маршрутов в папке /data |

### Внешние системы

| Система | Описание | Тип взаимодействия |
|---------|----------|-------------------|
| **Browser LocalStorage** | Хранение настроек, истории, пользовательских маршрутов | Чтение/запись через JavaScript API |
| **GitHub Pages** | Хостинг статического веб-приложения | HTTPS для доставки контента |
| **File System (data/)** | Файлы маршрутов в формате Markdown | Fetch API для загрузки |

```
                    │                     │                     │
```


#### Формула расчета

```javascript
// Нормы расхода электроэнергии
function calculateEnergyConsumption(data) {
    const axleLoad = data.trainWeight / data.axleCount;
    const coefficient = getEnergyCoefficient(
        locomotiveType, 
        axleLoad, 
        routeData
    );
    
    // Основная формула:
    // (Вес × Коэффициент × Расстояние) / 10000 / 100
    const energyConsumption = (
        data.trainWeight * 
        coefficient * 
        routeData.distance
    ) / 10000 / 100;
    
    return energyConsumption;
}
```

### Структуры данных

#### Маршрут (Route)

```javascript
{
    id: string,              // Уникальный идентификатор
    name: string,            // Название маршрута
    distance: number,        // Расстояние в км
    travelTime: number,      // Время в пути в часах
    description: string,     // Описание
```

```javascript
{
    type: string,            // Тип локомотива (vl10u, 2es6, etc.)
```

## DDD-анализ (Domain-Driven Design)

### Предметная область

**Домен:** Железнодорожные грузовые перевозки, расчет норм расхода энергии

### Ключевые понятия (Ubiquitous Language)

| Термин | Определение |
|--------|-------------|
| **Норма расхода** | Расчетное количество электроэнергии на поездку |
| **Нагрузка на ось** | Вес поезда, приходящийся на одну колёсную ось |
| **Коэффициент** | Поправочный коэффициент, зависящий от типа локомотива и нагрузки |
| **Маршрут** | Участок пути между двумя станциями с известным расстоянием |
| **Локомотив** | Тяговая единица (электровоз/тепловоз) для ведения поезда |
| **Условные вагоны** | Расчетное количество вагонов для определения длины состава |

### Сущности (Entities)

1. **Locomotive (Локомотив)**
   - Идентификатор: type
   - Атрибуты: name, length, tractionType
   - Инварианты: length > 0, type уникален

2. **Route (Маршрут)**
   - Идентификатор: id (генерируется из name)
   - Атрибуты: name, distance, travelTime, coefficients
   - Инварианты: distance > 0, coefficients не пуст для хотя бы одного типа локомотива

3. **Train (Поезд)**
   - Атрибуты: weight, axleCount, locomotives[], wagons[]
   - Инварианты: weight > 0, axleCount > 0, weight/axleCount в диапазоне 5-25 т/ось

### Объекты-значения (Value Objects)

1. **EnergyCoefficient**
   - Атрибуты: locomotiveType, axleLoad, value
   - Неизменяемый, определяется значением

2. **CalculationResult**
   - Атрибуты: energyConsumption, axleLoad, trainLength
   - Создаётся как результат расчета

### Агрегаты (Aggregates)

**Агрегат: Calculation**
- Корень: EnergyCalculator
- Сущности: Route, Locomotive, Train
- Границы: Один расчет нормы расхода

### Репозитории (Repositories)

1. **RouteRepository** (localStorage + file system)
   - Методы: getAll(), getById(), save(), delete()
   - Хранение: файлы /data/*.md + localStorage для пользовательских

2. **HistoryRepository** (localStorage)
   - Методы: getAll(), add(), clear()
   - Хранение: localStorage ключ 'rzdCalculatorHistory'

### Доменные сервисы (Domain Services)

1. **EnergyCalculationService**
   - Метод: calculate(route, train, locomotive)
   - Ответственность: Применение формулы расчета

2. **RouteValidationService**
   - Метод: validate(route)
   - Ответственность: Валидация данных маршрута

## Расширяемость

### Добавление нового локомотива

1. Добавить запись в `LOCOMOTIVE_DATA` в `data.js`:
```javascript
newLocomotive: {
    name: 'Новый Локомотив',
    length: 35,
    type: 'electric'
}
```

2. Добавить UI карточку в `index.html`

3. Добавить коэффициенты в файлы маршрутов при необходимости

### Добавление нового маршрута

1. Создать файл `data/new-route.md`:
```markdown
# Название маршрута

- XXX км
- XX,XX часа

| Нагрузка на ось | 6 | 7 | ... |
|-----------------|---|---|-----|
| ВЛ10У | ... | ... | ... |
| 2ЭС6 | ... | ... | ... |
```

2. Добавить имя файла в список `routeFiles` в `data.js`
